### Python 高频数据容器

> 除了 list、tuple、str、set、dict

Python 标准库中的 `collections` 模块提供了几种高效的专用容器，它们在实际开发中出现频率很高，尤其在数据处理、算法和性能优化场景。以下是比较高频的几种（基于官方文档和社区使用情况）：

| 数据容器              | 模块/来源              | 主要特点                                      | 高频应用场景                                      |
|-----------------------|-------------------------|-----------------------------------------------|---------------------------------------------------|
| **deque**            | `collections`          | 双端队列，支持 O(1) 的左右端 append/pop      | 队列/栈实现、滑动窗口、BFS 算法、消费者-生产者模型、日志缓冲 |
| **Counter**          | `collections`          | dict 子类，专用于计数，支持 most_common()     | 统计元素频率、词频分析、投票统计、TopK 问题       |
| **defaultdict**      | `collections`          | dict 子类，缺失键时自动提供默认值             | 分组统计、构建邻接表、树/图结构、避免 KeyError    |
| **namedtuple**       | `collections`          | 轻量级“具名元组”，像类但不可变，支持字段访问 | 配置对象、CSV/日志记录行、返回多个值、替代 dataclass 的轻量场景 |
| **OrderedDict**      | `collections`          | dict 子类，记住键插入顺序（Python 3.7+ dict 已有序，但仍有用） | LRU 缓存实现、序列化需保持顺序、旧代码兼容        |
| **ChainMap**         | `collections`          | 多个 dict 的逻辑视图，优先查找第一个          | 配置层级覆盖（命令行 > 环境变量 > 默认）、作用域链模拟 |
| **array**            | `array` 模块           | 同类型紧凑数组（类似 C 数组）                  | 大量数值存储、内存优化、科学计算初步阶段          |

### 说明
- 这些容器都是**内置标准库**的，无需额外安装。
- 在数据科学/机器学习中，**pandas.DataFrame** 和 **numpy.ndarray** 使用频率极高，但它们属于第三方库（pandas/numpy），不属于 Python 核心内置。
- 如果涉及更高级需求（如排序集合），社区有时用第三方如 `sortedcontainers`，但不属于高频内置。

这些容器能显著提升代码可读性和性能，推荐在项目中多用 `collections` 模块！
